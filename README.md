# Smart Loan AI

An end-to-end AI-powered financial application that serves as a loan eligibility pre-check tool with an intelligent chatbot. Built with a FastAPI/Python backend for machine learning and reinforcement learning inference, and an Android MVVM frontend.

## Features

*   **Loan Eligibility Prediction:** Uses trained Random Forest/XGBoost models to predict loan approval probability based on 10 demographic and financial features.
*   **AI Financial Chatbot:** Context-aware chatbot powered by the Gemini API (with robust rule-based fallbacks) for answering financial queries.
*   **Smart Recommendations:** A Reinforcement Learning (Q-Learning) engine that analyzes a user's financial profile and provides personalized strategies to improve eligibility.
*   **Budget & EMI Tools:** Calculates Debt-to-Income ratios, savings rates, financial health scores, and estimates EMI affordability.
*   **Admin Dashboard:** Tracks user metrics, prediction accuracy, and chat logs.

## Architecture

*   **Frontend:** Android Studio (Java), XML layouts, Material Design 3, MVVM, Retrofit for API calls.
*   **Backend:** Python 3.11, FastAPI, Pydantic, scikit-learn, Stable-Baselines3/Custom Q-Learning.
*   **Database:** Firebase Firestore (with an in-memory fallback mechanism for local testing).
*   **Authentication:** JWT + Firebase Auth.

## 🚀 Setup & Installation

### 1. Backend (FastAPI) Setup

1.  Navigate to the `backend` directory:
    ```bash
    cd backend
    ```
2.  Create a virtual environment and install dependencies:
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```
3.  Configure environment variables:
    *   Copy `.env.example` to `.env`
    *   Add your `JWT_SECRET_KEY` and (optionally) `GEMINI_API_KEY`.
    *   Place your Firebase `serviceAccountKey.json` inside `backend/app/firebase/`.
4.  Run the application:
    ```bash
    cd app
    uvicorn main:app --reload --host 0.0.0.0 --port 8080
    ```
    *The API will be available at `http://localhost:8080` and Swagger UI at `http://localhost:8080/docs`.*

### 2. Machine Learning Pipeline (Optional / Re-training)

The project comes with an end-to-end ML pipeline. To regenerate the synthetic data, run EDA, or retrain the models:

1.  **Generate Data:** `python eda/scripts/generate_sample_data.py`
2.  **Run EDA:** `python eda/scripts/generate_eda.py`
3.  **Train ML Model:** `python backend/app/ml/train_model.py`
4.  **Train RL Agent:** `python backend/app/rl/train_agent.py`
